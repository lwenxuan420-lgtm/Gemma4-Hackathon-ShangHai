## 二、方式 A · Fork & PR 步骤

1. **Fork 官方仓库**：
   `https://github.com/gdgshanghai/Gemma4-Hackathon-ShangHai`

2. **在 Fork 中创建提交目录**（赛道字母 = `A`）：
   ```
   /submissions/2026/A/Gemma-Production-Repair-Dashboard/
   ```
   - 把本文件夹的**全部内容**放进该目录。
   - 路径建议用纯英文 ASCII 名 `Gemma-Production-Repair-Dashboard`（GitHub 对中文 / 空格路径偶发异常，纯英文更稳；中文全名放 PR 标题里）。

3. **确认提交目录含官方三件套**：
   - [ ] `README.md`（含环境安装步骤）
   - [ ] `requirements.txt`
   - [ ] 核心源码（`backend/agent.py` + `backend/tools.py` + 三看板 HTML + 适配层 JS）

4. **发起 Pull Request**，标题严格按官方格式：
   ```
   [赛道A] Gemma 产线维修智能数据看板 - Cheng Repair AI Lab
   ```

5. **PR 提交后**：队长（本人）在官方「Gemma 4 开发者大赛-现场赛材料提交」表单填写现场赛材料。

## 三、提交前自检清单

### 核心要求（官方硬性）
- [ ] 核心代码含 Gemma 4 调用逻辑，且重点展示**原生函数调用**
      （`backend/agent.py`：`generate_content` 真调 + ```` ```tool_code ```` / ```` ```tool_output ```` 多轮循环）
- [ ] 演示视频时长 **≤ 5 分钟**（当前 4:50），展示核心功能 + 真实痛点
- [ ] 技术报告含**模型选型理由**（`技术报告.md` §3.1）+ **架构设计**（§3.2）

### 内容完整性
- [ ] `README.md` 有明确**环境安装步骤**（Python 3.10+ / `pip install -r requirements.txt` / 配 Key / 起前后端）
- [ ] `requirements.txt` 在根目录，依赖可一键安装
- [ ] 三个外链（仓库 / 在线演示 / 视频）均可访问，且与报名表单一致

### 安全与脱敏
- [ ] 不含真实 API Key（仅 `backend/.env.example` 占位；真实 Key 只在本地 `.env`，未入包）
- [ ] 不含真实 Webhook / 账号密码 / 内网 IP / 人员真实姓名
- [ ] 演示数据均为脱敏数据，不代表真实生产数据
